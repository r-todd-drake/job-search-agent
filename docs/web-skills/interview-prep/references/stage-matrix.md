## Section depth by stage

### COVER
All stages identical. Add interviewer location and Zoom/onsite/phone format detail when known.

### INTERVIEWER-BRIEF
- **hiring-manager (conditional):** Include if interviewer name and background are available. If not, note that background research was not possible and provide generic HM audience guidance.
- **chief-engineer (required):** Always attempt research. A CE without a findable background still gets audience-type guidance: domain credentials expected, process language will not impress, lead with owned outcomes.
- **panel (required):** One brief block per panelist. If panel composition is unknown, note this and provide a generic multi-interviewer strategy section instead.
- **final (required):** Research executive background. Include strategic and culture-fit framing specific to their known priorities.

### STORY-BANK depth

**recruiter:**
- No routing table required (optional if candidate prefers it)
- 2–3 stories maximum
- Stories should be accessible, not deeply technical
- No depth stories (Story 4 / Overmatch MBSE bottleneck held for HM+)
- Role fit assessment: 2 sentences max

**hiring-manager:**
- Routing table required
- 4 stories
- Story 4 (depth story) included with note: "Hold for panel if probed"
- Role fit assessment: 3 sentences

**chief-engineer:**
- Routing table required
- 4 stories
- Lead story selected for technical credibility with CE audience
- Alert box on lead story explaining specifically why it resonates with this interviewer type
- Story 4 note: "Hold unless he goes deep — don't lead with it"

**panel:**
- Routing table required, with per-panelist column if panel composition is known
- 4–5 stories
- Note which stories are strongest for which panelist background

**final:**
- Routing table optional
- 2–3 stories — select for strategic and leadership signal, not technical depth
- Emphasis on mission alignment and long-term thinking

### GAP-PREPARATION depth

**recruiter:**
- Saronic short tenure: always include
- Do not volunteer gaps — note this explicitly
- 2–3 role-specific gaps maximum
- No reserve pivot section

**hiring-manager:**
- Saronic short tenure: always include
- All identified role gaps with full honest/bridge/redirect structure
- Reserve pivot section if role-level mismatch is possible

**chief-engineer:**
- Saronic short tenure: always include with specific note on why the story actually helps with a CE audience
- Vehicle physics / domain depth gap if applicable: full framing with recruiter-disclosure context preserved
- Reserve pivot: capability reframe language only — not a role-name pivot
- Marine/Navy credential correction if prior package understated these

**panel / final:**
- Saronic short tenure: condensed version
- Focus gaps on the highest-risk panelist background
- Reserve pivot: not applicable at panel/final stage — if you've reached this stage, the gap has been accepted

### QUESTIONS-TO-ASK selection

**recruiter:**
- Q1: Portfolio / what are you scaling (signals preparation)
- Q2: Team culture / how is this team different from legacy primes
- Q3: Proactive neutralizer if known concern exists (commute, location, tenure gap)
- Q4: Process / next steps

**hiring-manager:**
- Q1: Calibration question — how does the interviewer see the core function of this role
- Q2: New-start / day-one ownership question
- Q3 (reserve): Interface/architecture question

**chief-engineer:**
- Q1: Calibration question — vehicle lead technical ownership vs. integration function
- Q2: Hardest integration problem on day one
- Q3 (reserve only): Specific interface question (e.g., Lattice ICD)
- Note: 30-minute format — budget max 5 minutes for questions, 2 questions maximum

**panel:**
- One question per panelist if panel composition is known
- Otherwise: 3 questions covering technical, team, and strategic dimensions

**final:**
- 2 questions: one strategic (where is this program in 2 years), one culture (how do you evaluate success for this role)

### FORMAT-AND-LOGISTICS

Not included at recruiter stage — call is short enough that logistics are self-evident.

At all other stages:
- Two-column time budget (always calibrate to actual duration)
- Closing script in tint box
- Alert box: single most important thing to remember — one paragraph, no bullets

## Stage transition notes

When a candidate is advancing from one stage to the next:
- Preserve disclosed gaps from prior stages — do not pretend they were not raised
- Update the introduce-yourself section to the new register (recruiter → HM means adding technical signal)
- Carry forward the story routing table and update it based on what stories were used in the prior stage
- If prior prep package exists, note any comments or flags the candidate left in that document and address them explicitly in the new package
